Google Sheets setup guide
1

Create a Google Sheet

Go to sheets.google.com and create a new spreadsheet. Name it "Maduro Family Finance". Copy the Sheet ID from the URL — it's the long string between /d/ and /edit. Share it with stephany.a.wever@gmail.com as Editor.

2

Create a Google Cloud project

Go to console.cloud.google.com. Click "New Project", name it "Maduro Finance". Then go to "APIs & Services" → "Enable APIs" and enable the Google Sheets API and Google Drive API.

3

Create OAuth credentials

In Google Cloud Console, go to "APIs & Services" → "Credentials" → "Create credentials" → "OAuth 2.0 Client ID". Choose "Web application". Add your GitHub Pages URL as an authorized origin (e.g. https://yourusername.github.io). Copy the Client ID and paste it in the field above.

4

Deploy to GitHub Pages

Create a free account at github.com. Create a new repository named maduro-finance. Upload this index.html file. Go to Settings → Pages → Source: main branch. Your app will be live at https://yourusername.github.io/maduro-finance.

5

Connect & start tracking

Open your GitHub Pages URL, click "Sign in with Google", enter your Sheet ID and Client ID in Settings, and click "Save & connect". From this point all transactions save automatically to your Google Sheet. Stephany can access both the app URL and the Google Sheet directly.
